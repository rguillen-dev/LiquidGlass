# LiquidGlass v1.2 — Release Plan

Theme: **reach native-API parity without breaking the architecture.** Today the
native path only ever calls `.glassEffect(.regular)` / `.regular.tint()`. v1.2
exposes the rest of the expressive surface and fixes the one real correctness bug
(fallback ignores accessibility). All changes route through the existing central
dispatch in `GlassRenderingModifier`. Mostly additive; no breaking changes.

Recommended scope: **v1.2 = additive parity + a11y fix.** Defer the native
navigation toolkit (button styles, ToolbarSpacer, tabViewBottomAccessory, sheet
zoom, retiring GlassTabBar internals) to a v2.0 "navigation toolkit" milestone.

---

## In scope for v1.2

### 1. `GlassVariant` (`.regular` / `.clear` / `.identity`)
Add a variant axis alongside `GlassStyle`, threaded through `.glass(...)`,
`GlassCard`, `GlassButton`, `GlassTabBar`.
- Native: maps to `Glass.regular` / `.clear` / `.identity`.
- Fallback: `.regular` = current tuning; `.clear` = lower material + lower tint
  opacity; `.identity` = passthrough (no background).
- `.identity` is also the accessibility off-switch (see #3).
- Default `.regular` so all existing call sites are unchanged.

### 2. `interactive:` parameter
Add `interactive: Bool = false` to `.glass(...)`, `GlassButton`, `GlassTabBar`.
- Native: `.regular.interactive()` (system scale/bounce/shimmer/illumination).
- Fallback: keep the current scale+opacity press animation (it's our stand-in).
- `GlassButton` should adopt this internally and default to interactive on 26.

### 3. Accessibility in the fallback path (the one real bug fix)
In `GlassRenderingModifier`, read `accessibilityReduceTransparency` and
`accessibilityReduceMotion` and degrade the FALLBACK only (native handles itself):
- Reduce Transparency → push material toward opaque, raise stroke contrast.
- Reduce Motion → disable the interactive press animation.
Add tests asserting the resolved fallback config changes when these are set.

### 4. `glassUnion(id:in:)` wrapper
Wrap `glassEffectUnion(id:namespace:)` under a NON-shadowing name (mirrors
`glassMorphID`). Native forwards; fallback is a no-op. Doc the same-type/same-shape
requirements.

### 5. `glassMorphTransition(_:)` wrapper
Wrap `glassEffectTransition(_:)` with our enum mirroring
`.identity / .matchedGeometry / .materialize`. Native forwards; fallback no-op.

### 6. Docs & hygiene
- **Rewrite README** to match shipped code (it still describes v1.0 — missing
  `GlassEffectContainer`, `glassMorphID`, `glassThemeTint`, `GlassTabBar`).
- Document tint-as-meaning and the glass-on-glass / container rule.
- Add the `.glassProminent + .circle` artifact note where relevant.
- Expand the DocC catalog (currently a single stub page) to cover variants,
  interactivity, morphing, union, and the fallback contract.
- CHANGELOG entry; bump to 1.2.0.

## Explicitly deferred to v2.0 (navigation toolkit)
- `.buttonStyle(.glass / .glassProminent)` wrappers + control sizes.
- `ToolbarSpacer`, `.sharedBackgroundVisibility`, badge helpers.
- `.tabViewBottomAccessory`, `.tabBarMinimizeBehavior`, search-role tab.
- Sheet zoom morph (`matchedTransitionSource` + `.navigationTransition(.zoom)`).
- Possibly wrap native `TabView` glass on 26 while keeping custom `GlassTabBar`
  as the 17–18 fallback.
- Optional design-token layer for scaling the system.

## Risks / watch-items
- `.clear` needs a dimming layer to stay legible — document, don't auto-apply.
- Native interactive hit-shape quirk with `RoundedRectangle` — prefer button style
  for buttons; note in docs.
- Can't visually verify native path without Xcode 26 + iOS 26 sim — call out in PRs.
- Keep defaults identical so v1.1 call sites don't change behavior.
